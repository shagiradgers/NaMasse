Описание проекта

Приложение для учета индекса массы тела, затрат калорий и БЖУ(белки, жиры, углеводы)
Задачи: сделать простой и не перегруженный дизайн, понятную программу для всех, ну и самое главное — это сделать полезную программу.
Моя главная цель — это написать программу, которая облегчит жизнь обычного работяги.

Описание классов
Window - наследует Qwidget, является основным классом, вызывается при запуске.
Calculation - наследует QDialog, в нем производятся получение данных и основные вычисления.
Preview - наследует QDialog, вызывается при нажатии btn_preview из класса Settings, выводит основную информацию по работе с приложением.
Faq - наследует QDialog, вызывается при нажатии btn_faq из класса Settings, выводит основную информацию о терминах используемых в программе.

Settings - наследует QWidget, вызывается при нажатии btn_settings из класса Window, выводит окно с 3 кнопками (btn_faq, btn_preview, btn_theme).

Recommendation  - наследует QDialog, вызывается при нажатии btn_rec из класса Window, выводит рекомендации для пользователя.


Класс Window
__init__ - загрузка дизайна и вызов нужных функций.
table_clicked - обработчик двойных нажатий на таблицу.
btn_pressed - обработчик нажатий на кнопки (btn_enter_data, btn_save_xls, btn_graphic, btn_rec, btn_settings). Если нажата btn_enter_data показывает dialog, если нажата btn_save_xls вызывает функцию save_xls, если нажата btn_graphic вызывает функцию draw_graphic, передавая main_data, если нажата btn_settings показывает settings, если нажата btn_rec вызывает функцию update_data принадлежащую классу Recommendation и показывает dialog_rec.
del_item - удаление строки из базы данных с последующим вызовом draw_table. Вызывается из функции table_clicked при двойном нажатии на таблицу и в выборе: ‘Удалить строчку’.
change_item - заменяет в строке n прежние данные на новые, полученные из класса Calculation. Вызывается из класса Calculation при двойном нажатии на таблицу в выборе: ‘Изменить входные данные’.
draw_table - отрисовка таблицы с данными. Вызывается из функций: del_item, get_data.
add_to_table - запись данных в sql таблицу. Вызывается из функции calculate принадлежащей классу Calculation.
get_data - получение данных из sql таблицы. Вызывается из функций: add_to_table, __init__.
save_xls - сохранение данных из sql таблицы в xlsx файл. Вызывается из функции btn_pressed при нажатии btn_save_xls.
draw_graphic - отрисовка графика данных из sql таблицы (запись графика производится в xlsx файл). Вызывается из btn_pressed при нажатии btn_graphic.

recommendation - получение рекомендаций для user’a вызывается: из класса Recommendation, из функции draw_graphic, из функции btn_pressed.

	Описание кнопок в классе Window
btn_enter_data (Ввести данные) - при нажатии показывает экземпляр класса Calculation

btn_save_xls (Сохранить в таблицу) - при нажатии вызывает функцию save_xls, которая сохраняет данные из sql таблицы в xlsx файл

btn_graphic (Построить график) - при нажатии вызывает функцию draw_graphic, которая строит график в xlsx таблице использую данные из sql таблицы

btn_rec (Рекомендации) - при нажатии вызывает функцию recommendation, которая показывает список рекомендаций для пользователя

btn_settings (Настройки) - при нажатии показывает экземпляр класса Settings

Класс Calculation
__init__ - загрузка дизайна и вызов нужных функций.
accepted -  сохраняет полученные данные в переменную (вызывается если нажата кнопка ‘ok’  в диалоговом окне).
calculate - производит все вычисления, если mode = 1 передает все вычисления в функцию add_to_table, принадлежащую классу Window. Если mode = любому другому числу, передает все вычисления в функцию change_item принадлежащую классу Window

Описание работы кнопок в классе Calculation
dialogButtonBox - при нажатии  ‘ok’ вызывает функция accepted

Класс  Settings
__init__ - загрузка дизайна и вызов функции pressed при нажатии одной из кнопок (btn_faq, btn_preview, btn_theme).
pressed - вызывается при нажатии любой из кнопок (btn_faq, btn_preview, btn_theme). Если нажата btn_faq показывает dialog_faq, если btn_preview показывает dailog_preview, если btn_theme предлагает выбрать цветовую схему и вызывает change_style.
change_style - заменяет цвет в зависимости от переданного, вызывается из функции pressed.

btn_faq (FAQ) - при нажатии показывает экземпляр класса Faq
btn_preview (О программе) - при нажатии показывает экземпляр класса Preview
btn_theme (Поменять тему) - при нажатии показывает диалоговое окно, где предлагает пользователю выбрать цветовую схему приложения. При нажатии ‘ok’ вызывает функцию change_style
